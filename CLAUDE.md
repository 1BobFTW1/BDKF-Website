# CLAUDE.md — Bund der Katzenfreunde (BDKF) Website

You are building the website for **Bund der Katzenfreunde** (bdkf.de), a German cat-lovers'
association. Target audience: urban singles, 30–55, design-aware. The brand voice is
**professional with a wink** — disciplined typography, playful copy, one cat mascot.
All site copy is **German**.

This file is the single source of truth for the brand system. It was developed and
stress-tested across posters, flyers, brochures, business cards, and Figma poster
comps before this website. Follow it exactly; the rules below exist because earlier
drafts broke without them.

---

## 1. The brand system (non-negotiable rules)

The identity is generated by ONE geometric rule:

> The letters **B, D, K, F** sit on the four corners of one rectangle.
> Collapsed to its minimum, the rectangle is the **Mark** (a 2×2 letter block).
> Opened to the edges of a format, it is the **Frame** (letters in the four corners).
> The Mark and the Frame are the same object at two sizes — never invent a third lockup.

### Hard laws (breaking any of these breaks the brand)

1. **Quadrant identity:** B is ONLY ever top-left, D top-right, K bottom-left,
   F bottom-right. Never reorder. Reading order spells BDKF in a Z-pattern.
2. **All four letters are always present and fully visible.** Never crop, cover,
   overlap, or substitute a letter (e.g. the cat must never stand in for the K —
   we tried it, it destroys the acronym for new viewers). A nav logo does NOT
   count as a corner letter: if a section uses the Frame, it gets all four letters
   in addition to any nav Mark.
3. **All four letters share one size and one weight** within a composition.
4. **Exclusion zones:** each corner letter's bounding box + 0.5 × letter-height
   padding on all sides is a no-go area. No text, images, or the cat may enter it.
5. **The cat never touches a letter.** It lives inside the content area
   (the space the letters frame). Story: the letters are the *Freunde* standing
   at the corners; the cat is the *Katze*, safe in the space they hold.

### Measurement system (all sizes derive from one unit)

- **U** = 1% of the format's SHORTER edge. In CSS this is exactly `1cqmin`
  (container query units) or `1svmin` at viewport level. Use container units.
- Margin **M = 6U** — identical on all four sides, measured, never eyeballed.
- Corner letter cap height **L = 10U**. (Font-size ≈ 14U gives ≈10U caps for most
  grotesques; verify optically with the real font.)
- Content starts **21U** from the top (M + L + 0.5L zone padding) and ends 21U
  from the bottom, EXCEPT in the horizontal band between the two corner zones,
  where content (typically the cat) may extend lower.
- **Frame minimum size:** shorter edge ≥ 600 px. Below that (e.g. mobile
  viewports), use the Mark instead of the Frame — see §5.

### The Mark (2×2 block)

- Grid: `B D` on row 1, `K F` on row 2. Column gap 0.16× cap height,
  row gap 0.10× cap height (rows sit optically tighter).
- Clear space ≥ 1× cap height on all sides. Minimum width on screen: 24 px.
- Colour: Ink on Paper or Paper on Ink only. Never amber, never grey.

## 2. Colour

| Token   | Hex       | Usage |
|---------|-----------|-------|
| Ink     | `#171717` | Letters, headlines, body on light, cat silhouette |
| Paper   | `#FDFCFA` | Backgrounds (warm off-white); letters/cat on dark |
| Katzenauge (amber) | `#E8A020` | THE accent. At most ONE amber element per view/section: one highlighted word, one CTA, or one date. If amber appears twice in one composition, that is one time too many. |
| Grau    | `#6E6D67` | Secondary text on light only. Never for the letters. |
| Grau (on Ink) | `#B9B8B2` | Secondary text on dark backgrounds. |

Sections may invert (Paper-on-Ink) freely; measurements never change.
The four letters are only ever Ink or Paper.

## 3. Typography — Forma DJR Deck

- **One family for everything: Forma DJR Deck** (David Jonathan Ross), served via
  Adobe Fonts (typekit). Ask the user for their Adobe Fonts kit ID / embed code and
  load it in the `<head>`. CSS family name is `forma-djr-deck`.
- Fallback stack: `"forma-djr-deck","Forma DJR Deck","Helvetica Neue",Helvetica,Arial,sans-serif`.
  Never substitute a rounded/decorative font to "add fun" — playfulness comes from
  the cat and the copy, never the typeface.
- Roles:
  - **Extra Bold** — the system letters (Mark + Frame) and hero headlines.
  - **Bold, uppercase, −2% tracking, 0.96–0.98 line-height** — headlines.
  - **Medium** — subheads and UI elements.
  - **Regular, 1.5 line-height** — body copy.
  - **Bold, uppercase, +14% tracking** — labels/eyebrows/captions.
- Max **three type sizes per view**. Headlines uppercase; body sentence case.
  Never letterspace headlines; only labels get tracking.

## 5. Website-specific translation of the system

This is how the print system maps to responsive web (worked out in §07 of the
brand guidelines):

- **Nav:** the 2×2 Mark, top-left, links right. The Mark is the logo everywhere.
- **Hero (desktop/tablet, shorter viewport edge ≥ 600 px):** the hero section is a
  Frame — all four letters pinned to ITS corners (the section's corners, not the
  viewport's), using container units: margin `6cqmin`, letter font-size `14cqmin`.
  Headline, intro, CTA, and cat live in the content area. The nav Mark above
  does NOT replace the hero's B — the hero carries all four letters itself.
- **Mobile (< 600 px shorter edge):** drop the hero Frame entirely (minimum-size
  rule); the nav Mark carries the brand. Do not render a cramped Frame.
- **Interior sections:** normal editorial layout, no corner letters — the Frame is
  a hero/cover device, not wallpaper. Use generous whitespace, the 3-size type
  scale, and at most one amber element per section.
- **Footer:** may be an inverted (Ink) Frame with the peek cat looking over its
  top edge, contact info in the content area — a nice bookend to the hero.
- **Favicon/app icon:** the Mark, Ink on Paper.
- Subtle scroll-triggered motion is welcome (the cat peeking in, letters settling)
  but must respect `prefers-reduced-motion`.

## 6. Site structure & content (German)

Pages/sections: **Start · Verein · Termine · Vermittlung (Adoption) · Mitglied werden · Kontakt**.
Adoption is the primary conversion goal; "Mitglied werden" is secondary. One amber
CTA per view — usually "Mitglied werden" or "Katze adoptieren".

## 7. Pre-flight checklist (run before calling any view done)

1. All four letters present, uncovered, fully inside their Frame?
2. B top-left, D top-right, K bottom-left, F bottom-right?
3. All four letters one size (L) and one weight (Extra Bold)?
4. Margin M identical on all four sides?
5. Every exclusion zone empty?
6. Everything in Forma DJR Deck, ≤ 3 sizes per view?
7. Amber appears at most once per view (outside cat eyes)?
8. Mobile shows the Mark, not a cramped Frame?
9. `prefers-reduced-motion` respected?

## 8. Tech preferences

- Ask the user before choosing a stack; default suggestion: a simple static or
  lightweight framework build (e.g. Astro or plain Vite + vanilla/React) — this is
  a content site for a Verein, not an app. German `lang="de"`, semantic HTML,
  WCAG AA contrast (Ink on Paper passes; check amber text sizes — `#E8A020` on
  Paper is large-text only), fast load, no heavy dependencies.
